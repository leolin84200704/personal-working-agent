## Phase 2: Scoring Engine

**Goal:** Implement deterministic importance scoring for all memory files.

### 2.1 Scoring formula

```python
import math
from datetime import datetime

def calculate_score(
    base_weight: float,       # from frontmatter (0.6 - 1.0)
    updated: datetime,        # last update date
    half_life: int,           # 30 for STM, 180 for LTM
    incoming_links: int,      # count of files linking TO this file
    normalization: float = 8.0
) -> float:
    days_since = (datetime.now() - updated).days
    recency = math.exp(-days_since / half_life)
    boost = 1.0 + (0.1 * incoming_links)
    return (base_weight * recency * boost) / normalization
```

### 2.2 Base weight mapping (adapted for LIS work context)

| Category | base_weight | Rationale |
|----------|-------------|-----------|
| emr_integration | 1.0 | Core business, highest impact |
| technical | 0.9 | Direct code knowledge |
| repo_patterns | 0.8 | Build/deploy patterns |
| pm_patterns | 0.7 | PM communication patterns |
| process | 0.6 | Workflow improvements |

### 2.3 New file: `src/memory/scorer.py`

```python
class MemoryScorer:
    def score_file(self, path: Path) -> float
    def score_all(self, directory: Path) -> dict[str, float]
    def rebuild_index(self, directory: Path, index_path: Path) -> None
```

Key behaviors:
- Reads YAML frontmatter from each file
- Calculates score using formula
- Writes updated `score:` back to frontmatter
- Rebuilds `_index.md` sorted by score descending

### 2.4 Thresholds

| Threshold | Score | Age | Action |
|-----------|-------|-----|--------|
| Active | >= 0.1 | any | Stays in current tier |
| Archive | < 0.1 | > 90 days | Move to `archive/` |
| Forget | < 0.05 | > 180 days | Delete permanently |
| Never auto-forget | any | any | `emr_integration` category (can be manually overridden) |

**Deliverables:**
- [x] `src/memory/scorer.py`
- [x] All files have updated `score:` in frontmatter after first scoring run
- [x] Thresholds defined and tested

**Risk:** Low. Scoring is read-only until Phase 3 uses it for consolidation decisions.

## Phase 4: Cross-Linking (Zettelkasten)

**Goal:** Add bidirectional references between related memory files.

### 4.1 Link storage

In YAML frontmatter:
```yaml
links:
  - emr-integration        # ID of linked file
  - VP-16270               # can link STM ↔ LTM
```

Links are always bidirectional: if A links to B, B must link to A.

### 4.2 Link discovery

During dreaming Phase 4 (Score & Reindex):
1. For each file, extract key topics/entities
2. Find other files with overlapping topics
3. Add bidirectional links if confidence > threshold
4. Linked files get `reference_boost` in scoring formula

### 4.3 New file: `src/memory/linker.py`

```python
class MemoryLinker:
    def discover_links(self, files: list[Path]) -> list[tuple[str, str]]

    def get_link_graph(self) -> dict[str, list[str]]
    def count_incoming_links(self, file_id: str) -> int
```

## Phase 5: Dreaming Pipeline & Automation

**Goal:** Automate the full 5-phase dreaming cycle via launchd.

### 5.1 Dream prompt: `scripts/dream.md`

A Claude Code prompt that executes all 5 phases:

```markdown
You are the LIS Code Agent's dreaming process. Execute these 5 phases:

## Phase 1: Orient
- Read short-term-memory/_index.md
- Read long-term-memory/_index.md
- Read all individual *.md files in both directories
- Note current date: {CURRENT_DATE}

## Phase 2: Gather Signal
For each STM file, classify as: completed, approaching, lasting_insight, overlap, stale

## Phase 3: Consolidate
Execute applicable operations: Extract, Merge, Update, Resolve, Promote, Archive, Forget

## Phase 4: Score & Reindex
- Recalculate importance scores for all active files
- Rebuild _index.md files (sorted by score descending)
- Discover and update cross-links

## Phase 5: Log
Write logs/dream-{DATE}.md with:
- Files scanned, archived, promoted, merged, updated
- Score statistics
- Memory stats (file counts per tier)
```

### 5.2 Runner script: `scripts/run-dream.sh`

```bash
#!/bin/bash
set -euo pipefail

AGENT_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
cd "$AGENT_ROOT"

DATE=$(date +%Y-%m-%d)
LOG_DIR="$AGENT_ROOT/logs"
mkdir -p "$LOG_DIR"

if [[ "${1:-}" == "--dry" ]]; then
    echo "DRY RUN: would execute dream pipeline"
    echo "claude -p \"\$(cat scripts/dream.md)\" --allowedTools Read,Write,Edit,Glob,Grep,Bash"
    exit 0
fi

claude -p "$(cat scripts/dream.md)" \
    --allowedTools "Read,Write,Edit,Glob,Grep,Bash" \
    2>&1 | tee "$LOG_DIR/launchd-stdout-$DATE.log"
```
