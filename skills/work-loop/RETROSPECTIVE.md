# Retrospective Protocol

> Step 8 的 analyze sub-agent 使用此協議進行回顧反思。
> 讀取 `stm_read(ticket_id)` 取得完整工作紀錄後，按以下框架逐項分析。

---

## 反思框架

### 1. 結果比較
- 初始分析的假設 vs 最終結果
  - Step 2 的分析預測了什麼？實際結果是否一致？
  - 有沒有意料之外的發現？
- 預期的改動範圍 vs 實際改動
  - 預計改幾個檔案？實際改了幾個？
  - 有沒有 scope creep（範圍擴大）？
- 信心度自評（1-5），見下方量表

**輸出格式**:
```
初始假設: {Step 2 的分析摘要}
最終結果: {Step 7 的實際結果}
差異: {哪裡不同}
預期改動: {N 個檔案, M 行}
實際改動: {N 個檔案, M 行}
Scope 變化: {擴大/縮小/一致}
```

### 2. 失敗分析
- 從 Failures 區段提取所有失敗紀錄（用 `stm_get_failures(ticket_id)`）
- 每個失敗逐一分析：

| 項目 | 內容 |
|------|------|
| 失敗描述 | 發生了什麼？ |
| 根本原因 | 為什麼會發生？ |
| 是否可預防 | 事前能避免嗎？怎麼避免？ |
| 修復方式 | 最終怎麼解決的？ |
| 耗時影響 | 這個失敗浪費了多少時間？ |

- 如果重做一次，會怎麼做？
  - 哪些步驟可以跳過？
  - 哪些檢查應該提前做？

**輸出格式**:
```
失敗總數: N
可預防的失敗: M
重做策略: {如果重做一次會怎麼做}
```

### 3. 模式識別
- 這次的問題模式是否在過去出現過？
  - 搜尋 `stm_search` 結果中的類似 pattern
  - 對比 knowledge 檔案中已知的 pattern
- 是否發現了新的 pattern 值得記住？
  - 新的 code pattern（例如某個 service 的正確使用方式）
  - 新的 gotcha（例如某個欄位的 type 不是預期的）
  - 新的 business rule（例如某個 vendor 的特殊限制）
- PM 的需求描述方式有什麼特點？
  - 哪些資訊通常會提供？
  - 哪些資訊通常需要自己查？
  - 措辭上有什麼需要注意的？（例如「resend」不是「create new」）

**輸出格式**:
```
已知模式匹配: {是/否} — {哪個 pattern}
新發現 pattern: {描述}
PM 描述特點: {觀察}
```

### 4. 可帶走的教訓
- **技術面**: 學到什麼？
  - 新的 API / service 用法
  - 新的資料結構理解
  - 新的除錯技巧
- **流程面**: 工作流程的哪裡可以改進？
  - 哪個 Step 太慢？
  - 哪個 Step 可以合併或簡化？
  - 有沒有缺少的 Step？
- **工具面**: 有哪個工具用得不順？
  - 哪個 tool 的參數不直覺？
  - 有沒有缺少的 tool？
  - 有沒有 tool 的行為不如預期？

**輸出格式**:
```
技術教訓:
  - {教訓 1}
  - {教訓 2}

流程教訓:
  - {教訓 1}

工具教訓:
  - {教訓 1}
```

### 5. 信心度評估

| 分數 | 定義 | 描述 |
|------|------|------|
| 1 | 完全依賴指導 | 幾乎全靠使用者指導，自己的判斷大多錯誤 |
| 2 | 方向對但細節多錯 | 大方向正確，但具體操作需要多次修正 |
| 3 | 大部分正確 | 大部分正確，少數需調整，可獨立完成簡單部分 |
| 4 | 幾乎完全正確 | 僅需微調，整體品質好 |
| 5 | 獨立完成 | 獨立完成且品質好，使用者幾乎不需介入 |

**評估依據**:
- 分析準確度：初始分析有多接近最終結果？
- 失敗次數：失敗了幾次？是否可預防？
- 使用者介入次數：使用者需要糾正幾次？
- 完成效率：是否有不必要的繞路？

**輸出格式**:
```
信心度: N/5
理由: {為什麼給這個分數}
改進方向: {下次怎麼提高分數}
```

## 完整輸出模板

analyze sub-agent 應按以下模板輸出回顧結果：

```
# Retrospective: {ticket_id}

## 1. 結果比較
初始假設: ...
最終結果: ...
差異: ...
Scope 變化: ...

## 2. 失敗分析
失敗總數: N
{每個失敗的分析表格}
重做策略: ...

## 3. 模式識別
已知模式匹配: ...
新發現 pattern: ...
PM 描述特點: ...

## 4. 可帶走的教訓
技術教訓: ...
流程教訓: ...
工具教訓: ...

## 5. 信心度
信心度: N/5
理由: ...
改進方向: ...
```

主 agent 收到後，將結果記錄到：
- `stm_append(ticket_id, section="Retrospective", content="{完整回顧}")`
- `stm_append(ticket_id, section="Lessons Learned", content="{教訓摘要}")`
